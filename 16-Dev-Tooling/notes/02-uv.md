## <a name="what-is-uv"></a>🤔 What is uv?

> **💡 Key Concept:** uv is an extremely fast Python package and project manager written in Rust. It replaces `pip`, `pip-tools`, `pipenv`, `poetry`, and `virtualenv` with a single tool — and runs 10-100x faster than pip.

It handles:
- Installing packages (`uv add`)
- Managing virtual environments (automatically)
- Locking dependencies (`uv.lock`)
- Running scripts (`uv run`)
- Installing Python itself (via `uv python install`)

## <a name="vs-pip--pipenv--poetry"></a>⚡ vs pip / pipenv / poetry

| Feature | pip | pipenv | poetry | uv |
|---------|-----|--------|--------|----|
| Speed | Slow | Slow | Medium | ⚡ Very fast |
| Lock file | ❌ (pip-tools) | ✅ | ✅ | ✅ |
| Virtual env | ❌ (manual) | ✅ | ✅ | ✅ auto |
| pyproject.toml | ❌ | ❌ | ✅ | ✅ |
| Install Python | ❌ | ❌ | ❌ | ✅ |
| Recommendation | Legacy | Avoid | Good | ✅ Use this |

## <a name="starting-a-project"></a>🚀 Starting a project

```bash
# New project from scratch
uv init my-project
cd my-project

# Initialize in an existing directory
uv init

# Install Python if needed (works alongside mise)
uv python install 3.13
```

This creates:

```
my-project/
├── pyproject.toml   # project metadata + dependencies
├── uv.lock          # exact locked versions
├── .python-version  # Python version pin
└── main.py
```

## <a name="commands"></a>🧰 Commands

```bash
# Add a dependency
uv add fastapi
uv add "django>=5.0"
uv add --dev pytest          # dev-only dependency
uv add --dev ruff mypy       # multiple at once

# Remove a dependency
uv remove fastapi

# Install all dependencies from lockfile
uv sync

# Install only production deps (no dev)
uv sync --no-dev

# Update all packages
uv lock --upgrade

# Update one package
uv lock --upgrade-package fastapi

# Run a script inside the project environment (no manual activation needed)
uv run python manage.py runserver
uv run pytest
uv run python main.py

# Run a one-off tool without installing it
uvx ruff check .
uvx black .

# Show installed packages
uv pip list

# Show dependency tree
uv tree
```

## <a name="pyprojecttoml"></a>📄 pyproject.toml

```toml
[project]
name = "my-project"
version = "0.1.0"
requires-python = ">=3.13"
dependencies = [
    "fastapi>=0.115.0",
    "uvicorn>=0.32.0",
    "sqlalchemy>=2.0.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=8.0.0",
    "ruff>=0.8.0",
    "mypy>=1.13.0",
]

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"
```

## <a name="virtual-environments"></a>🌍 Virtual environments

> **💡 Key Concept:** uv creates and manages a `.venv` in your project directory automatically — you never need to run `python -m venv` or `source .venv/bin/activate` manually.

```bash
# uv creates .venv automatically when you add/sync
uv add fastapi   # .venv is created if it doesn't exist

# To activate manually (rarely needed)
source .venv/bin/activate   # Linux/macOS
.venv\Scripts\activate      # Windows

# uv run handles activation for you — preferred
uv run python main.py       # ✅ no manual activation needed
```

```bash
# .gitignore
.venv/
__pycache__/
*.pyc
*.pyo
```

## <a name="good-practices"></a>✅ Good practices

```bash
# ✅ Commit both pyproject.toml and uv.lock
# pyproject.toml = what you need
# uv.lock = exact versions everyone installs

# ❌ Don't commit .venv — it's OS-specific
echo ".venv/" >> .gitignore

# ✅ Use uv run instead of activating manually
uv run python manage.py migrate

# ✅ Separate dev and prod dependencies
uv add fastapi                  # production
uv add --dev pytest ruff mypy   # development only

# ✅ Use uv sync in CI/CD
uv sync --no-dev               # only install prod deps

# ✅ Use mise to pin Python version, uv to manage packages
# .mise.toml → python = "3.13"
# pyproject.toml → requires-python = ">=3.13"
```

| Do | Don't |
|----|-------|
| Commit `pyproject.toml` + `uv.lock` | Commit `.venv/` |
| Use `uv run` to execute scripts | Manually activate venv |
| Separate dev deps with `--dev` | Put everything in main deps |
| Use `uv sync` in CI | Run `pip install` in CI |
| Let mise handle Python version | Manage Python version manually |
